Выполни сохранение состояния сессии перед переходом в новую.

1. Запусти `git status --short` — посмотри есть ли несохранённые изменения.
2. Если есть незакоммиченные изменения — спроси: "Закоммитить перед выходом?" и жди ответа.
3. Запиши файл `.claude/session_status.md` со следующей структурой:

```markdown
# Session Status — Dev — {дата и время}

## Что сделано в этой сессии
{краткий список завершённых задач}

## Текущее состояние
{общий статус проекта, что работает}

## Следующий шаг
{конкретная первая задача для новой сессии, одна фраза}

## Открытые вопросы
{что требует решения или уточнения}

## Версии
agent/{последний тег} · server/{последний тег} · cloud/{последний тег}
```

4. Выведи содержимое файла пользователю.
5. Скажи: "Сессия сохранена. Можешь открывать новую — контекст подхватится автоматически."
