Выполни сохранение состояния сессии перед переходом в новую.

1. Запусти `git status --short` в текущей папке — посмотри есть ли несохранённые изменения.
2. Если есть незакоммиченные изменения — спроси: "Релизить перед выходом?" и жди ответа.
3. Запиши файл `.claude/session_status.md` в текущей папке со следующей структурой:

```markdown
# Session Status — cloud — {дата и время}

## Что сделано в этой сессии
{краткий список завершённых задач}

## Текущее состояние
{что работает, что сломано, на каком месте остановились}

## Следующий шаг
{конкретная первая задача для новой сессии, одна фраза}

## Открытые вопросы
{что требует решения или уточнения}

## Версия
cloud/{последний тег из git}
```

4. Выведи содержимое файла пользователю.
5. Скажи: "Сессия сохранена. Можешь открывать новую — контекст подхватится автоматически."
